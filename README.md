# Motorola-Mobile-Sales-Dashboard

## Introduction  
As part of my data analytics training, I developed a **Motorola Mobile Sales Dashboard** using Microsoft Power BI. The purpose of this project was to transform raw mobile sales data into meaningful insights through interactive visualizations. By working on this project end‑to‑end — from **data cleaning and preparation** to **dashboard design and storytelling** — I practiced how business intelligence tools can support real‑world decision‑making.  

## Project Workflow  

1. **Data Preparation:**  
   - Imported raw sales data into Power BI.  
   - Cleaned inconsistencies, standardized column names, and formatted dates.  
   - Created calculated measures using DAX (e.g., average sales per transaction, payment method percentages).  

2. **Data Modeling:**  
   - Defined relationships between tables (sales, customers, products).  
   - Built hierarchies for geography and time to enable drill‑down analysis.  

3. **Visualization Design:**  
   - Selected appropriate chart types for each metric (map for geography, line for trends, pie for distribution).  
   - Structured the dashboard layout for clarity and logical flow.  
   - Added slicers and filters for interactivity (mobile model, payment method, brand, day, month).  

## Dashboard Highlights  

- **City‑wise Sales (Map):** Identified regional demand patterns and growth opportunities.  
- **Monthly Trends (Line Chart):** Revealed seasonal fluctuations, peaks, and dips in sales.  
- **Customer Ratings (Bar Chart):** Showed satisfaction levels, highlighting proportions of 5‑star vs lower ratings.  
- **Payment Methods (Pie Chart):** Analyzed consumer preferences across UPI, Debit Card, Cash, and Credit Card.  
- **Mobile Model Comparisons (Bar Chart):** Highlighted top‑performing devices, guiding product strategy.  

Together, these visuals created a **multi‑dimensional view of sales performance**, enabling stakeholders to explore data from different perspectives.
